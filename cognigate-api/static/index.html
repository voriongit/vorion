Quick Start

Get started with Cognigate in seconds

# 1. Parse an intent
curl -X POST https://cognigate.dev/v1/intent \
  -H "Content-Type: application/json" \
  -d '{"entity_id": "my-agent", "raw_input": "Send email to john@example.com"}'

# 2. Enforce the decision
curl -X POST https://cognigate.dev/v1/enforce \
  -H "Content-Type: application/json" \
  -d '{"intent_id": "int_...", "entity_id": "my-agent"}'

# 3. Get the proof receipt
curl https://cognigate.dev/v1/proof/prf_...

The Governance Stack

Three layers working together to ensure AI agents operate within bounds

INTENT ? The INTENT layer parses natural language or structured input to understand what an AI agent wants to do before it does it. It normalizes goals into a structured, policy-checkable format.

Normalize and validate agent intentions before execution. Extracts what the agent wants to do, translates it into structured policy-checkable format, and ensures clarity of purpose.

POST /v1/intent

ENFORCE ? The ENFORCE layer evaluates normalized intents against BASIS policies in real-time. It returns ALLOW, DENY, or ESCALATE with detailed reasoning for every decision.

Evaluate intentions against BASIS policies in real-time. Checks permissions, constraints, and safety boundaries. Returns ALLOW, DENY, or ESCALATE with detailed reasoning.

POST /v1/enforce

PROOF ? The PROOF layer generates cryptographic receipts for every governance decision. These immutable records enable full accountability and auditability.

Generate and verify cryptographic evidence of all governance decisions. Creates immutable audit trails, enables accountability, and provides receipts for every action.

POST /v1/proof
