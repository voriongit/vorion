## Contribute

BASIS welcomes all types of contributions:

### Code
- Reference implementation improvements
- SDK development
- Tooling and integrations
- Bug fixes

### Documentation
- Guides and tutorials
- Translations
- Examples and use cases

### Community
- Answer questions
- Moderate discussions
- Organize events

### Research
- Trust algorithms
- Security analysis
- Regulatory mapping

## Contributor Ladder

```
LURKER
  │  Join Discord, read docs
  ▼
PARTICIPANT
  │  Ask questions, give feedback
  ▼
CONTRIBUTOR
  │  Submit PRs, write content
  ▼
CORE CONTRIBUTOR
  │  Regular commits, lead initiatives
  ▼
MAINTAINER
  │  Own subsystems, review PRs
  ▼
STEWARD
  │  Shape direction, represent movement
```

## Governance

BASIS is evolving toward community governance:

### Phase 1 (Now)
Core team makes decisions with community input

### Phase 2
Contributors join governance council

### Phase 3
Foundation formed with elected board

### Phase 4
Full DAO governance for token holders

## RFC Process

Propose changes to the standard:

```
1. IDEA      → GitHub Discussion
2. DRAFT     → Formal RFC proposal
3. REVIEW    → Community feedback
4. DECISION  → Accept or reject
5. IMPLEMENT → Update spec & code
```

[Submit an RFC →](https://github.com/voriongit/basis-standard/discussions)

## Code of Conduct

We're committed to a welcoming community:

- **Be respectful** — Treat everyone with dignity
- **Be constructive** — Focus on ideas, not people
- **Be inclusive** — Welcome diverse perspectives
- **Be patient** — Help newcomers learn

Full [Code of Conduct](https://github.com/voriongit/basis-standard/blob/main/CODE_OF_CONDUCT.md)
